# Keras Architecture Recovery Process  
### Learning-by-Doing Architecture Roadmap  

This project follows a progressive architecture recovery methodology.
Each week combines **software architecture theory learning**
with **hands-on exploration of the Keras framework**.

The recovery workflow is organized around three core architectural views:

1. Module View  
   Understanding how Keras source code is logically decomposed
   into packages, layers, models, optimizers, datasets, etc.

2. Component-and-Connector View  
   Understanding runtime behavior such as:
   - Model training execution flow
   - Interaction between layers, optimizers, and loss functions
   - Backend computation pipeline (TensorFlow engine)

3. Allocation View  
   Understanding how Keras software elements map onto:
   - Development environment
   - Python runtime
   - Hardware (CPU/GPU)
   - External infrastructure (GitHub, package ecosystem)
