# Keras Architecture Recovery Process

This project follows a learning-by-doing philosophy.

Each week the architecture of Keras will be progressively recovered
through small runnable experiments and architectural documentation.

The recovery workflow follows three main architectural viewpoints:

1. Module View  
   Understanding how the source code is logically decomposed.

2. Component-and-Connector View  
   Understanding runtime interactions between major elements.

3. Allocation View  
   Understanding how software elements map to environment,
   infrastructure, and development ecosystem.

Artifacts produced weekly:
- Module map updates
- C&C diagrams
- Allocation insights
- Small AI experiments
- QA architecture analyses
