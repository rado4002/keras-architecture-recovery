# Recovering the Architecture of Keras  
 

## 📌 Project Context

This repository contains a **solo academic project** focused on recovering and documenting the software architecture of the open-source deep learning framework **Keras**.

The project is conducted as part of a first course in **Software Architecture**.

At the same time, the author has **little to no prior background in Artificial Intelligence**, therefore the project also serves as a structured introduction to AI concepts through practical exploration.

---

## 🎯 Project Goals

The main objective is to **understand and reconstruct the architecture of Keras** through progressive investigation and experimentation.

Specific goals include:

- Recover the **Module View** (logical code organization)
- Recover the **Component-and-Connector View** (runtime interactions)
- Recover the **Allocation View** (mapping to environment and infrastructure)
- Learn fundamental AI and deep learning concepts
- Apply software architecture theory to a real industrial-scale system

---

## 🧠 Learning Philosophy

This project follows a **learning-by-doing approach**.

Each week:

- A small runnable experiment is performed
- Architectural insights are documented
- Diagrams are progressively refined
- Understanding evolves from beginner level to structured expertise

The focus is not only on documentation, but also on **developing architectural thinking skills**.

## 🔬 Methodology

Architecture recovery is performed using:

- Source code exploration
- Controlled experiments with Keras
- Documentation synthesis
- Iterative refinement of architectural views

The project evolves in **weekly increments**, gradually increasing depth and precision.

## 📖 Motivation

Understanding the architecture of large open-source systems is a critical skill for software engineers.

Keras is chosen because it represents:

- A real-world industrial AI framework
- A modular layered architecture
- A system integrating mathematical computation, APIs, and backend engines

This project aims to bridge the gap between **software architecture theory and modern AI systems**.
