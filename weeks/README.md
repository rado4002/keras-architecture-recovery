# Weekly Workspace Layout

This folder organizes the architecture recovery work by week.

## Standard weekly structure

Each `weekXX` folder should contain:

- `docs/` — weekly logs, recovery notes, and summaries
- `diagrams/` — Mermaid diagrams viewable directly on GitHub
- `experiments/` — runnable scripts used for evidence collection
- `interpretation/` — architectural meaning, responsibilities, and patterns
- `notes/` — personal working notes
- `qa/` — question/answer and validation artifacts
- `slides/` — presentation materials

## Naming convention

- Use zero-padded week names: `week01`, `week02`, `week03`, ...
- Keep files descriptive and stable (avoid spaces/special characters in names).
